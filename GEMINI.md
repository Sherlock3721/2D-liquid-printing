# Laboratorní 2D Tisk Kapalin (G-Code Editor)

This project is a specialized Python-based application designed for laboratory research, enabling precise 2D printing of liquids onto substrates like microscope slides. It serves as both a G-code editor and a printer controller.

## Project Overview

The application allows researchers to:
- **Import Designs:** Support for SVG, DXF, and raw G-code.
- **Configure Lab Substrates:** Presets for standard lab glass (76x26mm), FTO glass, and custom dimensions.
- **Multiplex Printing:** Automatically layout multiple samples across the printer bed with defined spacing.
- **Tečky (tečkování):** (Ve vývoji) Přesné dávkování kapek z vektorových bodů/kružnic.
- **Research-Grade Parameters:** Control extrusion in microliters per millimeter (µl/mm), Z-offsets for thin-film deposition, and complex infill styles.
- **Direct Control:** Integrated serial terminal and worker for real-time printer communication (Marlin/Prusa compatible) with instant stop (M410) and pause (M601/M602) support.
- **Data Persistence:** Exports G-code with embedded JSON metadata for perfect state reproduction and CSV protocols for experimental logging.

## Core Architecture

### GUI (PyQt6)
- `main.py`: The entry point and main window orchestration.
- `gui/left_panel.py`: Global parameters, Z-height calculations, and printer controls.
- `gui/graphics_view.py`: Interactive 2D/3D preview of the print bed and toolpaths.
- `gui/right_panel.py`: Individual overrides for per-slide parameter variations.
- `gui/settings.py`: Dialog for editing machine limits, offsets, and loop G-codes.

### Core Logic
- `core/logic.py`: High-level orchestration, file loading, and multiplex layout mathematics.
- `core/gcode_generator.py`: Generates final G-code strings. Applies Z-heights and transformations.
- `core/vector_slicer.py`: Converts SVG/DXF vectors into printable paths with custom infills.
- `core/tecky.py`: (Plánováno/WIP) Logika pro bodové dávkování kapalin.
- `core/csv_exporter.py`: Generates experimental protocols.

### Communication
- `printer_com.py`: Multi-threaded serial communication handler for non-blocking printer control.

## ⚠️ CRITICAL PROJECT RULES (AI AGENT INSTRUCTIONS) ⚠️
Do NOT alter these core mechanics without explicit user instruction:

1. **Coordinate System & Layout:** - The printer's physical Home (0,0) is Front-Left.
   - However, the multiplex layout algorithm **starts at the Front-Right** corner (`bed_max_x - start_offset_x - slide_w`) and populates towards the Left (X-) and Back (Y+).
2. **Z-Height & PINDA Calibration:**
   - The printer uses a manual PINDA probe trick. The probe is physically lowered to the glass surface, making physical Z=0 align with the top of the glass.
   - Do NOT add glass thickness to the G-code `Z` coordinate. The G-code should only contain the requested `z_offset` (layer height).
3. **Configuration Flow:**
   - Base settings (bed size, offsets, start G-codes) are strictly loaded from `settings_default.json` and saved to `settings.json`.
   - Do NOT hardcode physical dimensions or layout offsets in `config.py`.

- **UI:** PyQt6
- **Communication:** pyserial

## Building and Running
```bash
# Create and activate virtual environment
python -m venv venv
source venv/bin/activate  # On Linux / Arch (fish)

# Install dependencies
pip install -r requirements.txt

# Execution
python main.py
```

## Development Conventions
- **Language:** The codebase uses a mix of English and Czech (e.g., GCodeApp vs zpracovat_soubor). Keep new UI elements in Czech.
- **Metadata:** G-code files generated by the tool include a header block (`; --- EDITOR METADATA ---`). Do not remove this, as it is used to restore the UI state when reloading files.
- **Hardware:** Designed primarily for Prusa/Marlin-based 3D printers modified for liquid dispensing.

## Configuration
- `settings.json`: Critical printer bed dimensions and initialization G-code.
- `config.py`: Physical substrate dimensions and default process parameters.
